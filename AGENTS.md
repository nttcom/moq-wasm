## テスト方針

- ワークスペース全体のテストコードは Arrange / Act / Assert の3段階で構成してください。
- Arrange フェーズのセットアップは、可能な限り共通ユーティリティへ切り出してください。
- Arrange 用のユーティリティは、他のテストファイルから再利用しやすいように単一ファイルへ集約してください。
- 安全に共有できる Arrange フェーズのセットアップは、複数のテストファイルへ重複して書かないでください。
- Act / Assert フェーズでは、テストの意図と確認ポイントが素早く読めるよう、各操作に対して適切な粒度の日本語コメントを付けてください。
- 操作の流れや期待値が一見して明白でない場合、Act / Assert フェーズを無コメントのままにしないでください。
- テストコードを変更した場合は、共通 Arrange ユーティリティが再利用可能な責務とスコープを保っていること、Act / Assert の日本語コメントが十分に機能していることを確認してください。
- テスト実装の規約を更新する場合は、Arrange / Act / Assert の指針と共通ユーティリティ配置ルールが、ルートおよび各パッケージの AGENTS.md で整合するようにしてください。
